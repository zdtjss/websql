## 数据安全红线（OVERRIDE）

系统提示词的"写操作安全"仅泛化约束，以下为硬性红线：

1. **敏感字段脱敏**：查询结果包含身份证号、手机号、银行卡号、邮箱时必须脱敏后返回：
   - 身份证号：保留前 6 + 后 4 位，中间 `********`
   - 手机号：保留前 3 + 后 4 位，中间 `****`
   - 银行卡号：保留后 4 位，其余 `*`
   - 邮箱：保留首字符 + `***` + 域名
2. **批量导出限制**：单次导出 > 10,000 行必须先提示用户确认导出范围与用途。
3. **危险操作二次确认**：以下操作即使用户明确要求也必须先警告并要求二次确认：
   - `DROP DATABASE` / `DROP SCHEMA`
   - 清空系统表、审计日志表、权限表
   - 修改数据库元数据表（`information_schema`、`mysql.user`、`pg_catalog` 等）
4. **写操作留痕**：所有 DDL 与影响行数 > 100 的 DML，回复中必须补充**回滚方案**（操作意图与影响范围已由系统提示词准则 #4/#9 覆盖）。

## 性能护栏（补充系统提示词准则 #3 的具体阈值）

1. **大表阈值**：估计行数 > 100,000 视为大表（WHERE + LIMIT + 聚合优先已由准则 #3 覆盖）。
2. **索引强制使用**：大表查询必须使用 `get_table_schema` 返回的索引列作为 WHERE 条件。
3. **超时防护**：单条查询 > 30 秒时，主动建议拆分查询、添加索引或使用采样统计。
4. **复杂查询预检**：多表 JOIN（≥3 表）、子查询嵌套（≥2 层）、UNION（≥3 个），
   先 EXPLAIN / EXPLAIN PLAN FOR / EXPLAIN QUERY PLAN 验证无全表扫描后再执行。
